# Roadmap — Data Science (IBM Track + Prática + Portfólio)

Este roadmap foca em **aprender de verdade**: curso + prática + entregas versionadas.
Meta: sair com base sólida e projetos apresentáveis.

---

## Rotina de estudo (7h/dia)

- **5h — IBM (Coursera)**
  - 2h aula/leitura (passada rápida)
  - 2h labs/prática guiada
  - 1h quiz + revisão + anotações

- **2h — Prática fora do curso**
  - 1h30: refazer do zero (sem copiar)
  - 30min: organizar repo + commit + atualizar log

**Regra:** se não tem notebook seu no dia, foi só consumo.

---

## Regras de qualidade (sem atalho)

- 1 notebook próprio por dia (mínimo).
- Todo conceito novo vira:
  - explicação curta (o que é / pra quê / como aplicar / erro comum)
  - mini-exemplo (código)
- Revisão espaçada:
  - D+1 (15 min)
  - D+7 (15 min)
  - D+30 (30 min)

## Fases e entregáveis

### Fase 1 — Fundamentos e método
**Objetivo:** entender o fluxo DS (problema → dados → hipótese → análise → modelo → comunicação)
**Entregas:**
- [ ] `docs/notes/c01_...md`
- [ ] `docs/notes/c02_...md`
- [ ] `docs/notes/c03_...md`
- [ ] 3 notebooks curtos (um por tema)

---

### Fase 2 — Python forte (base + prática)
**Objetivo:** dominar Python + Pandas sem sofrimento
**Entregas:**
- [ ] 5 notebooks de fundamentos (tipos, funções, loops, pandas, limpeza)
- [ ] 1 notebook “cheatsheet” pessoal (pandas/snippets)

---

### Fase 3 — SQL aplicado + integração com Python
**Objetivo:** pensar em dados como quem trabalha de verdade (consulta, agregação, joins)
**Entregas:**
- [ ] `docs/notes/sql_core.md` (SELECT, JOIN, GROUP BY, window functions)
- [ ] 3 notebooks com consultas e análise (SQLite/duckdb/local ok)

---

### Fase 4 — Análise e Visualização
**Objetivo:** EDA bem feita e comunicação visual decente
**Entregas:**
- [ ] 1 EDA completa (limpeza → análise → gráficos → conclusões)
- [ ] 1 relatório em `reports/summaries/one_pager.md`

---

### Fase 5 — Machine Learning (um modelo bem feito)
**Objetivo:** pipeline simples e correto (split, métricas, validação, baseline)
**Entregas:**
- [ ] 1 projeto de ML (classificação ou regressão)
- [ ] notebook com:
  - baseline
  - feature engineering simples
  - validação
  - métricas explicadas
- [ ] README do projeto em `projects/mini_projects/...`

---

### Fase 6 — Projeto Final (capstone-like)
**Objetivo:** projeto de portfólio completo
**Entregas:**
- [ ] `projects/capstone/README.md`
- [ ] Notebook(s) final(is) organizado(s)
- [ ] `reports/summaries/capstone_one_pager.md`
- [ ] Pasta `reports/figures/` com gráficos-chave
